Descripción

Sistema de Detección de Similitud entre Sílabos de Instituciones de Educación Superior (IES) del Ecuador

Este sistema está basado en la aplicación de tecnologías semánticas y minería de textos

ANTECEDENTES

  • Los cursos universitarios son gestionados mediante sistemas basados en sílabos, como es el caso de las Instituciones de Educación Superior (IES) del Ecuador.
  • No existe una estructura normalizada de sílabos para las universidades, generando variedad de formatos y modelos de datos.
  • Esta situación afecta procesos académicos, entre ellos, la movilidad estudiantil y/o convalidación de créditos entre IES.
  • Tiempo elevado en tareas de comparación entre sílabos (tareas manuales)

PROPUESTA

  • Aplicación de tecnologías semánticas y de minería de textos.
  • Desarrollar de un modelo común de datos de sílabos para todas las IES del Ecuador. (Ontología)
  • Crear el repositorio común de sílabos de IES del Ecuador.
  • Desarrollar servicios que permitan la detección automática de similitudes entre sílabos de las IES del Ecuador.
  • Desarrollar un sistema informático para la consulta y detección de similitud entre sílabos.


Arquitectura Propuesta


EXTRACCIÓN DE DATOS

  • Cada IES maneja su estructura de datos de sílabos.
  • Un proceso ETL para la extracción de datos por cada IES.
  • Se crea una BD Temporal por cada IES con datos preprocesados (manejo de campos vacíos, nulos, duplicados, etc)

MODELO DE DATOS (ONTOLOGÍA)

  • Se reutiliza modelos para lograr estandarización, integración e interoperabilidad
  • Creación de la ontología utilizando las siguientes herramientas de software: - NEON: Metodología de desarrollo. - PROTÉGÉ: Creación, reutilización, etc. - OOPS!: Validación. - WIDOCO: Publicación en formato html.
  • El modelo desarrollado esta basado en las siguientes ontologías: AIISO y DOCO: contenidos académicos. - BIBO: fuentes bibliográficas. - FOAF: datos de personas (profesores, coordinadores, etc).

GENERACIÓN DE RDF (RDF-IZATION)

  • Es el proceso de transformar los datos de bases de datos tradicionales a formato RDF (Basado en el modelo ontológico)
  • Proceso ETL: Utilizando LOD-GF framework para Pentaho Data Integration
  • Los Plugins de LOD-GF permiten: - Leer la ontología - Configurar mappings - Generar RDF - Cargar RDF a un Triplestore

DETECCIÓN DE SIMILITUDES

  • Se aplican ténicas, metodologías y herramientas de minería de textos
  • Se combinan diferentes técnicas para el análisis
  • Se obtiene un porcentaje de similitud entre pares de sílabos

Sistema de detección de similitud entre sílabos

  • Página web que permite navegar y consultar información del respositorio semántico de sílabos y detectar similitudes
